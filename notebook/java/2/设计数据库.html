# 设计数据库

思考场景，在纸上画出所有模块，设计成单表，然后在思考表与表中的关系，确立 1对1、1对多、多对多

# 二、表结构

主要是为表设计字段

需要考虑表与表之间的关系，如下：

1对多表，设计到两张表 例如： tb_user 与 tb_user_type, tb_user.type_id = tb_user_type.id

多对多表，则需要三张表，人（user） 与 房子（house）, 一个人可以有多间房子，一间房子可以住多个人。

则需要 tb_user 人基础表 、tb_house 房子基础表 以及 tb_user_house 用户 房子 关系表

tb_user.id = tb_user_house.user_id

tb_house.id = tb_user_house.house.id

注意：我们其实发现 tb_user_house.user_id 和 tb_user_house.house_id 应该不能有多次，其实可以设计一个唯一索引这两个字段（UNIQUE KEY）可以避免造成不必要的重复

UNIQUE KEY 'user_id_house_id_index' ('user_id', 'house_id') USING BTREE

# 三、唯一索引

UNIQUE KEY 'order_no_index' ('order_no') USING BTREE

# 四、单索引以及组合索引

加快查询速度

通过常规的查询条件，来判断那些字段需要添加所有，一般需要查询的单字段需要单索引，组合查询字段，则需要组合索引

KEY 'order_no_index' (order_no) USING BTREE

KEY 'order_no_user_id_index' (order_no, user_id) USING BTREE

create_time, update_time 这些记录时间，对于数据排查，数据统计都有很重要的作用
