# 支付宝支付

商户应用私钥 （开发者生成）JAVA 开发需要 支付请求发起

商户应用公钥（开发者生成）

支付宝公钥（支付宝提供） JAVA 开发需要 支付会结果回调

支付宝私钥（看不到）

RSA 非对称加密：公钥私钥是一对

发起支付的过程：商户 -> 签名（商户应用私钥） -> 请求支付宝 -> 商户通过（商户应用公钥）验证签名

异步通知

支付宝（支付宝私钥）-> 商户（支付宝公钥验证签名）

RSA 签名 不等同于加密

通常是用 AES 加密解密

大公司 修改数据库 格式的时候，需要 SQL 操作的，不能随便数据库软件添加，为何？

因为使用例如 Navicate 等软件修改表结构，是不会有记录的，不利于追踪数据库的修改记录，查错，理解

应该有一个专门记录数据库的修改记录的地方，每次的修改数据库结构，都应该记录在他SQL文档当中, 今后初始化项目时 ，也可以直接直接 SQL 文档初始化。保证了线上环境与本地文档的高度统一。这才是数据库维护的标准

alter table mall_pay_info add pay_amount decimal(20,2) NOT NULL COMMENT '支付金额' after platform_status
