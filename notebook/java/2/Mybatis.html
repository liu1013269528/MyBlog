# Mybatis

# 一、Mybatis VS Jpa

Jpa 比较简洁

约定大于配置，Jpa 封装了很多接口，简单的操作都被封装好了，基本上不需要实现。

Mybatis 是现状；Jpa 是趋势。具体还是需要看各个公司目前的情况来使用

# 1. 构架实体类

一般是根据数据表字段，类型，设置成小驼峰的属性名，并设置 getter / setter 方法

package com.imooc.mall.pojo;

import java.util.Date;

/**
 * po(persistent object) 持久化对象
 * pojo(plain ordinary java object) 简单 JAVA 对象
 */

public class Category {

    private Integer id;

    private Integer parentId;

    private String name;

    private Integer status;

    private Integer sortOrder;

    private Date createTime;

    private Date updateTime;

    public Integer getId() {
        return id;
    }

    public void setId(Integer id) {
        this.id = id;
    }

    public Integer getParentId() {
        return parentId;
    }

    public void setParentId(Integer parentId) {
        this.parentId = parentId;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public Integer getStatus() {
        return status;
    }

    public void setStatus(Integer status) {
        this.status = status;
    }

    public Integer getSortOrder() {
        return sortOrder;
    }

    public void setSortOrder(Integer sortOrder) {
        this.sortOrder = sortOrder;
    }

    public Date getCreateTime() {
        return createTime;
    }

    public void setCreateTime(Date createTime) {
        this.createTime = createTime;
    }

    public Date getUpdateTime() {
        return updateTime;
    }

    public void setUpdateTime(Date updateTime) {
        this.updateTime = updateTime;
    }

    @Override
    public String toString() {
        return "Category{" +
                "id=" + id +
                ", parentId=" + parentId +
                ", name='" + name + '\'' +
                ", status=" + status +
                ", sortOrder=" + sortOrder +
                ", createTime=" + createTime +
                ", updateTime=" + updateTime +
                '}';
    }
}

# 2. 在 dao 包下，创建 Mapper 接口

package com.imooc.mall.dao;

import com.imooc.mall.pojo.Category;
import org.apache.ibatis.annotations.Param;
import org.apache.ibatis.annotations.Select;

//@Mapper 直接在 入口程序 使用 MapperScan 可以节省注解
public interface CategoryMapper {
    @Select("select  * from mall_category where id = #{id}")
    Category findById(@Param("id") Integer id);
}

通常，没个 Mapper 都需要接口都需要 @Mapper 注解，否则不能使用如：@Select 等注解，其实可以统一在 入口 Application 文件中添加

@SpringBootApplication
// 统一设置 Mapper，记得把包名填好
@MapperScan(basePackages = "com.imooc.mall.dao")
public class MallApplication {
    public static void main(String[] args) {
        SpringApplication.run(MallApplication.class, args);
    }
}

# 4.调用

Category category = categoryMapper.findById(100001);
System.out.println(category.toString());

# 注意点

由于我们的实体类，其实是用的驼峰方式来写的，这是符合 JAVA 代码规范，但是你会发现，数据库明明有数据，但是读取的为 null，这需要单独做一些配置信息

# 增加配置，实现字段支持驼峰
mybatis:
  configuration:
    map-underscore-to-camel-case: true

# 三、 Mybatis Xml 方式

Xml 现在大多数公司还是在用

目标：熟练使用

需要注意的点：就是 Mapper 接口如何 与 xml 文件的对应关系，通过增加路径配置，依赖了包含包名等

# 主要步骤

  1. 配置文件，配置 Mybatis xml 路径，否则会找不到 ##Mapper.xml 文件

mybatis:
  configuration:
    map-underscore-to-camel-case: true # 增加配置，实现字段支持驼峰
  mapper-locations: classpath:mapper/*.xml # 指定 Mybatis xml 文件路径

在 dao 层写，Mybatis 方法

public interface CategoryMapper {
  	// 找指定分类
    Category queryById(Integer id);
}

熟悉 Mybatis xml 格式，可以去官网找 Mybatis 文档

路径 ： resource/mapper/CategoryMapper.xml 注意 xml 文件名与 dao 层的 Mapper 接口要保持一致，这是约定

# 第一阶段：

使用 select * 其实很不好，将来修改数据库字段，可能暴露不安全的字段

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.imooc.mall.dao.CategoryMapper">
    <select id="queryById" resultType="com.imooc.mall.pojo.Category">
        select * from mall_category where id = #{id}
    </select>
</mapper>

# 第二个阶段

选择出需要的字段 代替 *

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.imooc.mall.dao.CategoryMapper">
    <select id="queryById" resultType="com.imooc.mall.pojo.Category">
        <!-- 尽量不要 select * -->
        select
        id, parent_id, name, status, sort_order, create_time, update_time
        from mall_category
        where id = #{id}
    </select>
</mapper>

# 第三个阶段

所有的字段可能不止使用一次，则需要提取出来，作为公用字段

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.imooc.mall.dao.CategoryMapper">

    <!-- 共用字段 -->
    <sql id="BaseColumnList">
        id, parent_id, name, status, sort_order, create_time, update_time
    </sql>

    <select id="queryById" resultType="com.imooc.mall.pojo.Category">
        <!-- 尽量不要 select * -->
        select
        <include refid="BaseColumnList" />
        from mall_category
        where id = #{id}
    </select>
</mapper>

注意 namespace 和 resultType 对应关系，一定要对应准了，才能找指定的 Mapper 和 返回类型，参数 #{id} 与 com.imooc.mall.dao.CategoryMapper 参数一直
