# 一、UML 定义

统一建模语言（Unified Modeling Language, UML）非专利第三代建模语言。

# 特点

  • UML 是一种开放的方法
  • 用户说明、可视化、构建和编写一个正在开发的面向对象的、软件密集型系统的制品（模型，源代码）的开放方法
  • UML 展现了一系列最佳工程实践，这些实践对大规模、复杂系统进行建模方面，特别是在软件架构层次被证明有效

# 分类

# 结构式图形

强调的式系统式建模

分类名 包含
静态图 类图*、对象图、包图
实现图 组件图、部署图
剖面图
符合结构图

# 行为式图形

强调系统建模中出发的事件

- 活动图
- 状态图
- 用例图

# 交互式图形

属于行为式图形的子集合，强调系统模型中资料流程

- 通信图
- 交互概述图
- 时序图*
- 时间图

# UML 类图

Class Diagram: 用于表示类、接口、实例等之间相互的静态关系

类图不只有类

# 记忆技巧

# UML 箭头方向：由 子类 指向 父类

理解：子类 extends 父类，子类明确父类定义，父类却不知道子类的定义

# 空心三角箭头：继承-类 或 实现-接口

实线：继承关系，is a 关系，扩展目的，不虚，很结实

虚线：实现关系

空心菱形： 聚合， 整体与局部的关系，具有独立生命周期，has a 关系，弱关系

实心菱形：部分与整体的关系，强关系，具有相同的生命周期， contains a， 强关系

# 常见数字表达与含义

表达式 描述
0..1 0或1个实例
0..* 0或多个实例
1..1 1个实例
1 只有一个实例
1..* 至少有一个实例

# UML 时序图

Sequence Diagram: 是现实对象之间交互的图，这些对象按照时间顺序排列；

时序图包含元素：对象、生命线 控制焦点（Focus of control）、消息（message） 等
